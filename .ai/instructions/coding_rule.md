# コーディング・製造ルール

## 概要
プロジェクトの実装品質を維持し、AIと開発者が混乱なく作業を進めるためのルールを規定します。特に、ディレクトリ構成の整合性を最優先事項とします。

## 1. ディレクトリ構成とパッケージ管理
- **標準構成の厳守**: Javaソースコードは必ず Spring Boot の標準的なディレクトリ構成（`src/main/java/com/aimealplan/{package}/`）に配置すること。
- **パス指定の禁止事項**:
  - 実装クラスや設定ファイルをプロジェクトルート直下に配置してはならない。
  - ファイル作成・移動時に、ローカルマシンの絶対パス（例: `C:\Work\...`）を基準にしたディレクトリ（`c/` や `Work/`）をリポジトリ内に生成してはならない。
- **パッケージ宣言の整合性**:
  - ソースファイルの `package` 宣言は、配置ディレクトリと完全に一致させること。
  - 例: `src/main/java/com/aimealplan/entity/User.java` のパッケージは `package com.aimealplan.entity;` とする。

## 2. 実装方針（Java / Spring Boot）
- **Lombokの活用**:
  - ボイラープレートコード削減のため、`@Data`, `@NoArgsConstructor`, `@AllArgsConstructor`, `@Builder` を積極的に使用する。
- **JPA / Entity**:
  - `jakarta.persistence` パッケージを使用する（Spring Boot 3系準拠）。
  - テーブル名は複数形（例: `users`）、カラム名はスネークケース（例: `created_at`）で明示的に指定する。
- **Repository**:
  - `JpaRepository` を継承し、クエリメソッドを活用する。

## 3. 製造時のチェックリスト
AIはコードを生成・提案する前に、以下の項目を自律的に確認すること：
1. [ ] **配置パスの再確認**: 提案するファイルの `+++` パスが `src/main/java/...` で始まっているか？
2. [ ] **不要ディレクトリの防止**: `c/` や `Work/` といった階層がパスに含まれていないか？
3. [ ] **既存ルールの遵守**: `.ai/instructions/github_rule.md` および `docs_rule.md` と整合しているか？

## 4. トラブル発生時の対応
- もし誤った階層にファイルが作成された場合、即座にイシューを作成し、本ドキュメントおよび `github_rule.md` に基づいて修正（Refactor）を行うこと。
- 修正後は `README.md` の「進捗状況」を更新し、構成が正常化したことを明記すること。

## 目的
「正しい場所にあるべきコードがある」状態を維持することで、AIによるコンテキスト理解の精度を高め、デプロイやビルドの失敗を未然に防ぎます。
