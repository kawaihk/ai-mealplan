# README 更新ルール

## 概要
AI-Native開発において、AIがプロジェクトの最新状態を常に正しく認識できるようにするため、作業内容のサマリーを `README.md` に蓄積することをルール化します。

## ルール内容
1. **更新タイミング**: 各セッションの終了時、または重要な機能実装・設計変更が完了したタイミングで必ず実行すること。
2. **更新箇所**: `README.md` の「2. プロジェクトの進捗状況」セクション、または必要に応じて「今後の予定」セクション。
3. **記載項目**:
   - 実施した作業の簡潔な概要。
   - 新規作成・変更した主要なファイルやディレクトリ。
   - **ディレクトリ構成ルールの遵守確認結果（例：レビューファイルが正しく `docs/reviews/` に配置されているか）。**
   - 決定した技術的方針や修正事項。

## 4. ファイル配置の最終チェック
`README.md` を更新する前に、AIは以下の項目を自律的に確認しなければならない：
- **レビュー結果の配置**: すべての `REV-*.md` ファイルが `docs/reviews/` 配下に存在し、ルート直下や `.ai/instructions/` などに誤って作成されていないか。
- **不要な階層の混入**: 絶対パス指定のミスにより、プロジェクトルートに `c/` や `Work/` といった意図しない階層が生成されていないか。
- **ルールとの整合性**: `.ai/instructions/docs_rule.md` 等で規定された配置ルールに違反していないか。

このルールを遵守することで、開発のトレーサビリティを確保し、AIによるコンテキスト把握の精度を最大化します。
