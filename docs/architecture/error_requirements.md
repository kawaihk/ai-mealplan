# エラー要件 (ID 13)

## 1. エラーレスポンス形式
APIで異常が発生した場合、以下の標準フォーマットでレスポンスを返却する。

```json
{
  "timestamp": "ISO8601形式の日時",
  "status": "HTTPステータスコード",
  "error": "エラー名",
  "message": "ユーザー向けメッセージ",
  "path": "リクエストされたパス",
  "details": [
    {"field": "フィールド名", "message": "詳細メッセージ"}
  ]
}
```

## 2. HTTPステータスコードの利用指針
- **400 Bad Request**: バリデーションエラー、パラメータ不正。
- **401 Unauthorized**: 未認証状態でのアクセス。
- **403 Forbidden**: 権限不足（他ユーザーのデータ操作等）。
- **404 Not Found**: リソースが存在しない。
- **500 Internal Server Error**: 予期せぬシステム例外。
