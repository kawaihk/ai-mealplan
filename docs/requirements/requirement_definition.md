# 献立管理アプリ 要件定義書 (v2.0)

## 1. プロジェクト目的
利用者が1週間の献立を計画・管理し、健康状態や栄養バランス（PFC）を直感的に把握できるシステムを構築する。AIによる提案機能を視野に入れつつ、まずは堅牢なデータ管理と使い勝手の良いUIを実現する。

## 2. システムアーキテクチャ
- **Backend**: Spring Boot 3.x (Java)
- **Database**: PostgreSQL
- **Frontend**: HTML5, Vanilla JavaScript, Tailwind CSS (React等のライブラリは使用しない)
- **Rendering**: サーバーサイドレンダリング(Thymeleaf) または 静的HTML + REST API

## 3. 機能要件
### 3.1 献立管理機能
- **週間ビュー**: 月曜から日曜までの1週間をグリッド表示し、各曜日の「朝食・昼食・夕食」のスロットを表示する。
- **食事登録/編集**: 各スロットに対して、料理名（Title）、構成品目（MealItems）、タグ（Tags）を登録できる。
- **栄養素管理**: 1食ごとの総カロリー、タンパク質(P)、脂質(F)、炭水化物(C)を数値で管理する。
- **ドラッグ＆ドロップ**: 既存の献立スロットを別の曜日や時間帯へ移動・入れ替えできる（フロントエンド制御）。

### 3.2 栄養分析機能
- **PFCバランス可視化**: 各食事カードに、PFCの比率を示す色付きインジケータを表示する。
- **日次/週次集計**: 1日および1週間単位での総カロリーおよび栄養素の合計値を算出する。

### 3.3 データ連携
- **CSVエクスポート**: 計画した1週間の献立リストをCSV形式でダウンロードできる。
- **AI提案（フェーズ2）**: 空白のスロットに対して、過去の傾向や栄養バランスを考慮した献立をAIが提案する。

## 4. データドメイン (主要エンティティ)
`docs/architecture/domain_definition.md` に基づく。
- **MealPlan (週間献立)**: 日付と曜日を保持。
- **Meal (食事)**: タイプ（朝・昼・夕）、タイトル、栄養素情報を保持。
- **MealItem (構成品目)**: 1つの食事に含まれる具体的な品目（例：ご飯、納豆）。

## 5. UI/UX 仕様
`mocks/designs/wireframe.md` の設計を遵守する。
- **デザイン**: クリーンでモダンなエメラルドグリーンを基調とした配色。
- **レスポンシブ**: モバイル時は1列、デスクトップ時は7列のグリッド表示。
- **インタラクション**: 
    - 編集時はモーダルまたはインラインフォームを表示。
    - カードホバー時に詳細情報をポップアップ。

## 6. 非機能要件
### 6.1 データベース整合性
- `UNIQUE` 制約（1日・1タイプにつき1献立）によるデータの重複防止。
- 参照整合性（CASCADE DELETE）による、献立削除時の子要素（アイテム、タグ）の一括削除。

### 6.2 パフォーマンス
- インデックス設計（日付、外部キー）による高速な検索。
- 静的資産（Tailwind CSS）はCDNまたは効率的なキャッシュ戦略を利用。

## 7. 実装ロードマップ
1. **Phase 1**: DBスキーマ構築とSpring Data JPAによるエンティティ実装。
2. **Phase 2**: Thymeleaf/HTMLとVanilla JSによる週間ビューの構築。
3. **Phase 3**: 登録・編集APIの実装と、ドラッグ＆ドロップ等の動的フロントエンド機能の追加。
4. **Phase 4**: 栄養素集計ロジックとCSV出力機能の実装。
