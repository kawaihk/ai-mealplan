## 1. 概要

本ドキュメントは、AI-MealPlan アプリケーションで使用するすべてのメッセージを定義する要件定義資料です。
実装ガイドライン（ID-15）のメッセージ管理方針に基づき、4つのカテゴリーのメッセージを体系的に整理しています。

---

## 2. メッセージ管理の基本方針

### 2.1 命名規則

メッセージキーの命名規則：

```
{domain}.{message_type}.{specific_message}
```

**例：**
- `validation.error.required`
- `error.meal.not_found`
- `notification.success.created`
- `confirm.delete.meal`

### 2.2 階層構造

```
messages.properties
├── Validation Messages（バリデーション）
├── Error Messages（エラー）
├── Notification Messages（通知）
└── Dialog Messages（ダイアログ）
```

## 4. メッセージ実装時の留意点

### 4.1 パラメータ化

メッセージに動的な情報が必要な場合は、パラメータプレースホルダー `{0}`, `{1}`, `{2}` を使用します。

**例：**
```
validation.error.max_length=「{0}」は{1}文字以下である必要があります

// 使用時
String message = getMessage("validation.error.max_length", "献立名", 100);
// 結果: 「献立名」は100文字以下である必要があります
```

### 4.2 メッセージの一意性

各メッセージキーはアプリケーション全体で一意である必要があります。
同じ文言でも用途が異なる場合は、異なるキーを割り当てます。

### 4.3 メッセージの国際化対応

本要件では日本語メッセージを定義していますが、将来的な多言語対応を考慮して、messages.propertiesは以下の構成とします：

```
messages.properties       (デフォルト：日本語)
messages_en.properties    (英語)
messages_zh.properties    (中国語)
...
```

### 4.4 ログ出力

ユーザーに表示するメッセージと、ログ出力用のメッセージは分離します。
ログ用の詳細情報はログファイルに出力し、ユーザーには理解しやすいメッセージを表示します。
